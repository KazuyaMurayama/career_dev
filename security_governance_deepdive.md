## 1. 現在のスキルシート記載と「第一線レベル」の差

### 現在の記載（入門レベル）

```
セキュアLLM環境（Azure OpenAI 等）でのRAG／AIエージェント構築・本番運用の実績あり。
セキュリティ要件が厳しい大手企業でのAIプロダクト開発、データ分類・プロンプト管理・アクセス制御・
監査ログ設計、AI出力物の説明可能性担保（SHAP等）、生成AI利用時の個人情報・
機密情報の取扱ポリシー策定の経験を有します
```

### 第一線レベルの記載（目指すレベル）

```
「エンタープライズ向け生成AIシステムのセキュリティ設計から本番運用までを一気通貫で担える」

具体的には：
- OWASP LLM Top 10 を起点に自社システムへ脅威モデルをマッピングし、防御設計に落とせる
- Azure OpenAI 固有の全統制（Managed Identity / Private Endpoint / Content Safety）を実装できる
- RAG パイプラインの間接プロンプトインジェクション・テナント分離・ PII 漏洩を設計段階で排除できる
- NIST AI RMF / ISO 42001 / EU AI Act / 個人情報保護法の要件を列挙したうえで対応設計ができる
- 監査ログスキーマ・アラート・インシデント対応手順をゼロから設計した実績がある
- SHAP による ML モデルの説明可能性実装・モデル監査・ドリフト検知まで担えた実績がある
```

**差の本質:** 現在の記載は「何をやったか」を列挙しているにとどまる。第一線レベルは「なぜそれがリスクなのか」「どう設計したか」「なぜその判断か」を具体的な言葉で記述できる。

## 2. スキルシートの「セキュリティ・ガバナンス」セクション 改訂案

### 現行のスキルマトリクス記載

| スキル | 経験年数 | レベル | 主な用途 |
|---|---|---|---|
| セキュアLLM環境（Azure OpenAI 等） | 2年 | ★★★★☆ | セキュリティ要件が厳しい大手企業でのRAG／AIエージェント構築・本番運用 |
| データ分類・アクセス制御・監査ログ設計 | 2年 | ★★★★☆ | プロンプト管理、機密情報の取扱、運用定着に向けた統制設計 |
| AI出力物の説明可能性担保 | 6年 | ★★★★★ | SHAP等による説明可能AI実装、モデル監査、経営層向け意思決定支援 |
| 生成AI利用ポリシー策定 | 2年 | ★★★★☆ | エンタープライズ導入時の個人情報・機密情報の取扱ルール設計、AI活用ガイドライン策定 |

### 改訂案（第一線レベル向け深層化）

| スキル | 経験年数 | レベル | 主な用途 |
|---|---|---|---|
| **LLMセキュリティ設計**（Azure OpenAI） | 2年 | ★★★★☆ | OWASP LLM Top 10を基に脅威モデルを定義。Managed Identity + Private Endpointでネットワーク層を閉域化。Content Safety API + システムプロンプト多層防御によるプロンプトインジェクション対策。大手5社で本番運用まで導いた実績あり |
| **RAGパイプラインセキュリティ** | 2年 | ★★★★☆ | 間接プロンプトインジェクション対策（コンテキスト分離）、テナント分離（メタデータフィルタ強制）、PIIパイプライン（NER+Presidioによる取り込み/検索/出力の3段階マスキング） |
| **ガバナンスフレームワーク対応** | 2年 | ★★★★☆ | NIST AI RMFの4機能（GOVERN/MAP/MEASURE/MANAGE）とISO/IEC 42001条項に照らし合わせてポリシー・監査・改善サイクルを設計。EU AI Act・個人情報保護法対応も含む |
| **監査ログ・インシデント対応** | 2年 | ★★★★☆ | LLM固有のスキーマ（request_id/トークン数/コンテンツフィルター結果）を設計・WORM保存。プロンプトインジェクション/情報漏洩/エージェント超過行動の3シナリオ別プレイブックを整備 |
| **説明可能性・AIガバナンス** | 6年 | ★★★★★ | SHAPにTreeExplainerで個別判断根拠を自動生成。LLM出力はRAG根拠文書提示+類似度スコア表示でハルシネーションを定量管理。モデルカード整備まで一気通貫 6年実績 |

## 1. 現在のスキルシート記載と「第一線レベル」の差

### 現在の記載（入門レベル）

```
セキュアLLM環境（Azure OpenAI 等）でのRAG／AIエージェント構築・本番運用の実績あり。
セキュリティ要件が厳しい大手企業でのAIプロダクト開発、データ分類・プロンプト管理・アクセス制御・
監査ログ設計、AI出力物の説明可能性担保（SHAP等）、生成AI利用時の個人情報・
機密情報の取扱ポリシー策定の経験を有します
```

### 第一線レベルの記載（目指すレベル）

```
「エンタープライズ向け生成AIシステムのセキュリティ設計から本番運用までを一気通貫で担える」

具体的には：
- OWASP LLM Top 10 を起点に自社システムへ脅威モデルをマッピングし、防御設計に落とせる
- Azure OpenAI 固有の全統制（Managed Identity / Private Endpoint / Content Safety）を実装できる
- RAG パイプラインの間接プロンプトインジェクション・テナント分離・ PII 漏洩を設計段階で排除できる
- NIST AI RMF / ISO 42001 / EU AI Act / 個人情報保護法の要件を列挙したうえで対応設計ができる
- 監査ログスキーマ・アラート・インシデント対応手順をゼロから設計した実績がある
- SHAP による ML モデルの説明可能性実装・モデル監査・ドリフト検知まで担えた実績がある
```

**差の本質:** 現在の記載は「何をやったか」を列挙しているにとどまる。第一線レベルは「なぜそれがリスクなのか」「どう設計したか」「なぜその判断か」を具体的な言葉で記述できる。

## 2. スキルシートの「セキュリティ・ガバナンス」セクション 改訂案

### 現行のスキルマトリクス記載

| スキル | 経験年数 | レベル | 主な用途 |
|---|---|---|---|
| セキュアLLM環境（Azure OpenAI 等） | 2年 | ★★★★☆ | セキュリティ要件が厳しい大手企業でのRAG／AIエージェント構築・本番運用 |
| データ分類・アクセス制御・監査ログ設計 | 2年 | ★★★★☆ | プロンプト管理、機密情報の取扱、運用定着に向けた統制設計 |
| AI出力物の説明可能性担保 | 6年 | ★★★★★ | SHAP等による説明可能AI実装、モデル監査、経営層向け意思決定支援 |
| 生成AI利用ポリシー策定 | 2年 | ★★★★☆ | エンタープライズ導入時の個人情報・機密情報の取扱ルール設計、AI活用ガイドライン策定 |

### 改訂案（第一線レベル向け深層化）

| スキル | 経験年数 | レベル | 主な用途 |
|---|---|---|---|
| **LLMセキュリティ設計**（Azure OpenAI） | 2年 | ★★★★☆ | OWASP LLM Top 10を基に脅威モデルを定義。Managed Identity + Private Endpointでネットワーク層を閉域化。Content Safety API + システムプロンプト多層防御によるプロンプトインジェクション対策。大手5社で本番運用まで導いた実績あり |
| **RAGパイプラインセキュリティ** | 2年 | ★★★★☆ | 間接プロンプトインジェクション対策（コンテキスト分離）、テナント分離（メタデータフィルタ強制）、PIIパイプライン（NER+Presidioによる取り込み/検索/出力の3段階マスキング） |
| **ガバナンスフレームワーク対応** | 2年 | ★★★★☆ | NIST AI RMFの4機能（GOVERN/MAP/MEASURE/MANAGE）とISO/IEC 42001条項に照らし合わせてポリシー・監査・改善サイクルを設計。EU AI Act・個人情報保護法対応も含む |
| **監査ログ・インシデント対応** | 2年 | ★★★★☆ | LLM固有のスキーマ（request_id/トークン数/コンテンツフィルター結果）を設計・WORM保存。プロンプトインジェクション/情報漏洩/エージェント超過行動の3シナリオ別プレイブックを整備 |
| **説明可能性・AIガバナンス** | 6年 | ★★★★★ | SHAPにTreeExplainerで個別判断根拠を自動生成。LLM出力はRAG根拠文書提示+類似度スコア表示でハルシネーションを定量管理。モデルカード整備まで一気通貫 6年実績 |

## 3. 面談・商談での証明のしかた

### 3.1 30秒ピッチ（エージェント初回・スクリーニング）

```
「エンタープライズ向け生成AIのセキュリティ設計を一気通貫で担える数少ないフリーランスです。
 OWASP LLM Top 10 を起点とした脅威設計から、Azure OpenAI の Managed Identity・
 Private Endpoint・Content Safety の実装、RAGの間接インジェクション対策、
 NIST AI RMF に沿ったガバナンス設計まで、大手5社の本番運用実績があります。」
```

**ポイント:** 「設計から運用まで一気通貫」「大手5社本番運用」の2フレーズを必ず入れる。

### 3.2 2分ピッチ（エージェント面談・登録面談）

```
背景（20秒）:
「STANDARD 在籍中、AI 導入プロジェクトを5社以上担当する中で、
 セキュリティ要件が一番の壁になるケースが最多でした。
 そこで OWASP LLM Top 10・NIST AI RMF を独学で習得し、
 設計書・ガバナンスフレームワーク・インシデント対応手順まで自分で引き受けました。」

実績（40秒）:
「具体的には、金融系クライアントで Azure OpenAI を Private Endpoint + 
 Managed Identity で閉域化し、テナント分離（メタデータフィルタ強制）と
 PII 3段階マスキング（Presidio）を実装。
 監査ログは LLM 固有スキーマで設計・WORM 保存し、
 プロンプトインジェクション・情報漏洩・エージェント超過行動の3シナリオ別
 プレイブックも整備しました。」

差別化（20秒）:
「ML/DS の6年実績と LLM セキュリティの専門性を両方持つのが強みです。
 SHAP による説明可能性・モデルカード整備まで含め、
 AI ガバナンス全体を設計できる人材は市場にほとんどいません。」

クロージング（20秒）:
「稼働率25〜50%での並行参画が希望です。
 まず貴社の AI システム現況をお聞かせいただけますか？」
```

### 3.3 クライアント商談で出る質問と模範回答

| 質問 | 回答（STAR形式） |
|---|---|
| **「プロンプトインジェクション対策をどう実装しましたか？」** | 「金融系クライアントで（S）、ユーザー入力とシステムプロンプトが混在するリスクを特定し（T）、Content Safety API による入力フィルタ + システムプロンプトの構造化分離 + 出力の2段階サニタイズを実装しました（A）。その後6ヶ月間インシデントゼロで本番運用できています（R）。」 |
| **「NIST AI RMF を実務でどう使いましたか？」** | 「AI 利用ポリシー策定（S）で、GOVERN/MAP/MEASURE/MANAGE の4機能に沿ってタスクを整理し（T）、OWASP LLM Top 10 のリスクを MAP フェーズで自社システムにマッピング、MEASURE で Red Team テストとハルシネーション率を定量化しました（A）。経営層へのリスク報告フォーマットとして活用されています（R）。」 |
| **「Azure OpenAI の Private Endpoint 設定経験は？」** | 「はい。Managed Identity で認証・APIキー完全排除、VNet 内の Private Endpoint でパブリックアクセスを遮断、Japan East リージョンで国内完結させた構成を5社に展開しました。」 |
| **「個人情報保護法の AI 対応はどうしていますか？」** | 「Azure OpenAI への入力前に Presidio で PII を検出・マスキング、ログにも平文保存しない設計にしています。Microsoft との業務委託契約（第24条）締結確認と、プライバシーポリシーへの AI 活用明記もセットで対応します。」 |
| **「ISO 42001 対応経験は？」** | 「認証取得ではなく設計レベルですが、条項4・6・8・9・10 に対応するポリシー・監査スキーマ・改善サイクルを整備した実績があります。27001 との統合運用も設計しました。」 |

### 3.4 逆質問で「わかっている人」を印象付ける

商談・面談で使える逆質問例：

```
「現在のAIシステムはマルチテナントですか？
 テナント間のデータ分離はどのレイヤーで担保していますか？」

「RAGを使っている場合、取得ドキュメントの権限チェックは
 ベクトルDB側でやっていますか、それともアプリ層ですか？」

「監査ログに LLM へのプロンプト全文を保存していますか？
 WORM 保存でコンプライアンス要件に対応していますか？」

「AI の判断根拠を経営層やエンドユーザーに説明する必要がありますか？
 説明可能性の要件レベルを教えてもらえますか？」
```

**効果:** これらの質問を自然に出せること自体が、第一線レベルの証明になる。

### 4.2 エージェント別の使い分け

**Tier S（コンサルフリー・HiPro Tech）:**
- スキルシート + セキュリティ記載を前面に出す
- 「OWASP LLM Top 10 / NIST AI RMF 実装経験」を件名・冒頭に入れる
- 月額300万円（フル稼働換算）を主軸に提示

**Tier A（みらいワークス・フリーコンサルBiz）:**
- スキルシート送付 + 面談時に 2分ピッチを使う
- 稼働率25〜50%の並行参画前提であることを明示

**FLEXY（STAR案件ルート）:**
- 職務経歴書 + スキルシートをセット送付済み
- クライアント面談では3.3の Q&A から該当質問を想定して準備

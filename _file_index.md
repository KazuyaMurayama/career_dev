## 🔄 本インデックスの保守ルール

1. **新規ファイル追加時**: 該当セクションに追記、最終更新日を更新
2. **ファイル削除・廃止時**: 該当行を削除 or 「（アーカイブ）」マーク追加
3. **最新版マーカー（★）**: 新しい最新版ができたら、前の「★」を外し新しい方に付ける
4. **バージョン履歴**: 大きな変更はバージョン履歴テーブルに追記
5. **運用ブランチ**: **main 一本のみ**。作業ブランチは作成しない（詳細 [CLAUDE.md](https://github.com/KazuyaMurayama/career_dev/blob/main/CLAUDE.md)）

**責任者**: Claude Code がこれを常に最新に保つ。ユーザーから新しいファイルをコミット報告された場合、本 `_file_index.md` も同タイミングで更新する。
